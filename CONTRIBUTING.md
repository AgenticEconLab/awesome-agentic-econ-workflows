# Contributing to Awesome Agentic Research Workflows for Economics

Thank you for your interest in contributing to this curated list! This document provides guidelines for contributing workflows, tools, and resources.

## How to Contribute

### Reporting Issues

If you find a broken link, outdated information, or have suggestions for improvements, please open an issue.

### Adding Items

1. **Fork the repository**
2. **Create a branch** for your contribution
3. **Add your item** following the format guidelines below
4. **Submit a pull request** with a clear description

## Contribution Guidelines

### Workflow Contributions

When adding a workflow, please include:

- **Title**: Clear, descriptive name
- **Description**: Brief explanation of what the workflow does
- **Use Cases**: When to use this workflow
- **Requirements**: Dependencies, tools, or prerequisites
- **Link**: URL to the workflow (GitHub, documentation, etc.)
- **Author/Creator**: Attribution if applicable

**Format:**
```markdown
- **[Workflow Name](link)** - Brief description. Use cases: X, Y, Z. Requirements: Tool A, Tool B.
```

### Tool Contributions

When adding a tool or framework:

- **Name**: Official name of the tool
- **Description**: What it does and why it's useful for economics research
- **Link**: Official website or repository
- **License**: Open source license (if applicable)
- **Status**: Active maintenance status

**Format:**
```markdown
- **[Tool Name](link)** - Description. License: MIT/Apache/etc. Status: Active/Maintained.
```

### Resource Contributions

When adding papers, tutorials, or other resources:

- **Title**: Full title
- **Authors**: Author names
- **Type**: Paper/Tutorial/Guide/etc.
- **Link**: DOI, URL, or repository link
- **Year**: Publication year (if applicable)

**Format:**
```markdown
- **[Title](link)** - Authors (Year). Type: Paper/Tutorial/etc.
```

## Quality Standards

- **Relevance**: Items must be directly relevant to agentic workflows in economics research
- **Quality**: Only well-maintained, documented, and useful items
- **Uniqueness**: Avoid duplicates or near-duplicates
- **Accessibility**: Prefer open-source or freely accessible resources

## Code of Conduct

Please review our [Code of Conduct](CODE_OF_CONDUCT.md) before contributing. We are committed to providing a welcoming and inclusive environment.

## Review Process

1. All contributions will be reviewed by maintainers
2. Feedback may be requested before merging
3. Maintainers reserve the right to edit for clarity and consistency
4. Merged contributions will be attributed to the contributor

## Questions?

If you have questions about contributing, please open an issue with the `question` label.

Thank you for helping make this resource better for the economics research community!
